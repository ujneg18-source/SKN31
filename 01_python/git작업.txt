1. 프로젝트 디렉토리 생성
~\my_proj

2. 로컬 저장소(repository)생성
  - git init
3. 파일 생성
   - a.txt
4. git repository 상태 확인
   - git status
   - 빨강색 파일/디렉토리 -> stage에 올릴 대상.
5. stage에 올리기
   - git add 올릴 파일/디렉토리 목록
    - `git add . ` 현재 디렉토리의 모든 파일/디렉토리를 다 올린다.

6. git status
   - 초록색 -> commit 대상

7. commit
   - git commit -m "커밋 메세지"
   - git commit -m "first commit"

- 파일 생성/내용 변경
  - git status
  - git add .
  - git commit - m "메세지"
  - git log

- git log
  git log --oneline
   - commit 한 히스토리 로그를 출력


- 이전 commit으로 돌아가기

   - git  reset  --옵션  commit_id
   - git reset HEAD~1

- github.com

git remote add origin https://kgmyhGit@github.com/kgmyhGit/my_proj.git

- git remote -v

- git push -u origin main

- readme.md 파일 생성
  - 프로젝트/디렉토리의 내용 설명 파일
  - 디렉토리에 readme.md 파일이 있으면 github에서는 그 내용을 보여준다.
   
  1. git add .
  2. git commit -m "readme 추가"
  3. git push origin main
